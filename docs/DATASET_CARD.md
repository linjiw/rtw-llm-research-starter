# Dataset card: RTW Countdown

## Purpose

A small, deterministic, verifiable dataset for studying reward curriculum and harness-aware post-training in LLMs.

## Generation

Tasks are generated by sampling random arithmetic expression trees. Because the target is computed from the sampled expression, every task is solvable by construction.

The verifier is still used after generation to ensure the stored solution is valid.

## Fields

- `id`: unique example id.
- `split`: train, validation, test_in_dist, test_ood_long, or test_ood_division.
- `difficulty`: easy, medium, hard, ood_long, or ood_division.
- `numbers`: integer multiset.
- `target`: target integer.
- `allowed_ops`: allowed operators.
- `solution`: one valid solution.
- `prompt_low`: low-informativeness harness prompt.
- `prompt_mid`: medium-informativeness harness prompt.
- `prompt_high`: high-informativeness harness prompt.
- `prompt`: default training prompt, currently equal to `prompt_high`.
- `completion`: synthetic SFT completion.
- `metadata`: generation metadata.

## Intended use

- GRPO/RLOO post-training with verifier rewards.
- SFT warmup for output format.
- Harness-shift evaluation.
- OOD generalization evaluation.

## Not intended use

- Claiming broad mathematical reasoning capability.
- Comparing against frontier closed models as a benchmark.
- Safety or alignment conclusions beyond the narrow verifier setting.

## Known limitations

- Synthetic expressions may contain stylistic artifacts.
- The task is narrow and symbolic.
- Some examples may have easier alternative solutions than the stored solution.
- No natural language ambiguity is modeled.

## Integrity and protocol status

The deterministic v0.16 audit is tracked at
`docs/artifacts/countdown_legacy_v1_audit.json` and summarized in
`docs/V16_COUNTDOWN_DATASET_AUDIT_PLAN.md`.

The committed files are historical `countdown-legacy-v1` artifacts. Every
stored solution/completion passes the verifier, but 39 rows have fewer operands
than their declared difficulty, cross-split semantic leakage is present, and
stored prompts predate the current prompt functions. README/Makefile generation
counts also do not match the committed file counts. Preserve these files for
legacy reproduction only; they are not eligible for corrected-v2 experiments
or a fresh final-test claim.

The corrected successor is documented in
`docs/COUNTDOWN_V2_DATASET_CARD.md`. Its tracked audit is
`docs/artifacts/countdown_v2_audit.json`; it is eligible for new within-v2
experiments, while its final-test split remains sealed and unreleased.
