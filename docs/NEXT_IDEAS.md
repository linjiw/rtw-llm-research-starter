# Next research extensions

## 1. Joint task-reward curriculum

Extend the teacher action from only reward weights to:

```text
a_T = (difficulty distribution, harness level, auxiliary reward weights)
```

This combines GACL-style task curriculum with RTW-style reward curriculum.

## 2. Grounded synthetic task generation

Keep a reference distribution of real benchmark tasks and a synthetic generator. Alternate between them with a grounding probability ε.

In the Countdown setting, reference tasks can come from a held-out hand-curated set, while synthetic tasks are generated by expression trees.

## 3. Code harness

Move from Countdown to Python function synthesis:

- primary reward: hidden tests pass;
- auxiliary rewards: code parses, imports cleanly, visible tests pass, no forbidden calls, short patch;
- harness levels: problem only, problem + examples, problem + examples + visible tests + failure traces.

## 4. Harness-shift robustness

Train with one tool/schema/prompt style and test under perturbed harnesses:

- renamed fields;
- changed argument order;
- lower-informativeness prompt;
- stricter verifier;
- hidden tests replacing visible tests.

## 5. Learned teacher

After the heuristic RTW controller shows signal, replace it with:

- contextual bandit;
- PPO teacher;
- recurrent controller over reward history;
- Bayesian optimization over reward schedules.
